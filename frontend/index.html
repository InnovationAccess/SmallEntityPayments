MDM – Entity Name Normalization

How to Use MDM Name Normalization

  1. Search for entity names using boolean expressions in the search bar:
    • GOOG* — wildcard matches GOOGLE, GOOGLE LLC, GOOGLE INC, etc.
    • MICRO*+CORP — AND: names containing both MICRO... and CORP
    • APPLE+-INC — NOT: names with APPLE but without INC
  2. Copy a representative name by clicking the copy icon in the table. This sets the "canonical" name.
  3. Paste to associate variant names with the copied representative. Click the paste icon on each variant row.
  4. Address lookup: Click the address icon next to any name to see registered addresses, then search for other entities at those addresses.
  5. Delete: Click the trash icon to remove an existing name association.

Frequency shows how many times each name appears across patent assignments. Higher frequency names are typically better candidates for the representative name.

Search entity names across patent and assignment data using boolean expressions. Select a Representative Name (Copy), then associate variant names with it (Paste). Use + (AND), - (NOT), * (wildcard).

Boolean Query Builder

How to Use the Query Builder

  1. Select tables to query against: Patent File Wrapper, Assignments, and/or Maintenance Fees. Hold Ctrl to select multiple.
  2. Add conditions by clicking "+ Add Condition". Each condition has:
    • Field — the data column to filter on (e.g., applicant_name, filing_date, patent_number)
    • Operator — LIKE, =, >, <, BETWEEN, IN, etc.
    • Value — the search value (use % as wildcard in LIKE queries)
  3. AND / OR toggle controls how multiple conditions combine. AND returns rows matching all conditions; OR returns rows matching any.
  4. Entity name expansion: When you search by applicant or assignee name, the system automatically includes all known name variants from the MDM tab.
  5. Click "Execute Query" to run. Results appear in an interactive table with sortable columns.
  6. Patent numbers in results are clickable — hover to see the assignment chain for that patent.

Construct multi-condition queries across patent data, assignments, and maintenance fee records. Entity name searches automatically expand via name normalization.

AI Assistant – Natural Language Query

How to Use the AI Assistant

  1. Ask in plain English. Describe the data you need, for example:
    • "Show me all patents filed by Samsung that changed from small to large entity"
    • "Find the top 10 assignees by number of maintenance fee payments in 2023"
    • "Which patents filed after 2020 have the most forward citations?"
  2. Refine through conversation. The AI remembers context, so you can say things like "narrow that to just California applicants" or "add the filing date column".
  3. The AI generates SQL behind the scenes and runs it against all your patent data tables. Click the "Generated SQL" accordion to see the query.
  4. Results appear as interactive tables with sortable columns. Patent numbers are clickable for assignment chain lookup.

The AI has access to all tables: patents, assignments (normalized v4), maintenance fees, citations, continuity data, and prosecution transactions.

Chat with the AI about your patent data. Describe what you need, refine the query through conversation, then ask it to run when ready.

Entity Status Analytics

        
      

      

        
How to Use Entity Status Analytics

        
This tab analyzes entity status (Small / Micro / Large) based on prosecution event codes in the patent file wrapper, not the USPTO's entity_status field.

        
    
          
  1. Single Patent Lookup — Enter a patent number to see its full entity status timeline: every SMAL, BIG., and MICR event in chronological order, plus any status changes.
    2. 
          
  2. Conversion Search — Find patents that changed entity status. For example, search for patents that went from Small to Large:
            
      
              
    • Set "From Status" and "To Status"
      • 
              
    • Optionally filter by grant year range and applicant name
      • 
              
    • Results show the conversion date and surrounding events
      • 
            
    
          
    3. 
          
  3. Applicant Portfolio — Enter an applicant name to see their complete entity status breakdown: how many patents filed as small vs. large, conversion rates, and trends over time.
    4. 
        

        
Boolean name search: Use +term* syntax in applicant fields, then click Find to search for matching names. Example: +elect* +telecom* finds names containing both "elect" and "telecom". Use -term* to exclude.

        
Event codes: SMAL = Small Entity declaration, BIG. = Large Entity declaration, MICR = Micro Entity declaration.
